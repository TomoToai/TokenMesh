# TokenMesh V2 Tasks

> 基准：GitHub `TomoToai/TokenMesh` 当前 `main` 分支。V1 已完成注册登录、Chat、流式豆包调用和会话历史。

## 当前状态

- 技术栈：Next.js 单体应用、SQLite、JWT、bcrypt、火山方舟 Ark API。
- 已有页面：首页、注册、登录、Chat。
- 已有 API：认证、会话管理、流式 Chat。
- 当前重点：在不破坏 V1 可用体验的前提下，补齐可商业化的 API 网关、计费和联网能力设计。

## P0：线上安全与稳定性

- 将 `JWT_SECRET` 设为生产必填，禁止使用默认值进入生产环境。
- 为登录和注册接口增加基础限流。
- 为 Chat 请求增加消息长度限制和历史上下文截断。
- 统一 API 错误格式，避免把上游完整错误原文直接暴露给用户。
- 增加最小化 smoke test：注册、登录、创建对话、发送消息。

## P1：API 网关商业闭环

- 新增 API Key 数据表和创建 / 列表 / 禁用接口。
- 新增 OpenAI-compatible `POST /api/v1/chat/completions`。
- 将当前页面 Chat 的 `/api/chat` 与 API 网关复用同一模型调用服务。
- 记录每次请求的 token usage、模型、状态、成本和错误码。
- 增加余额账户和扣费流水。

## P2：国内支付优先

- 先接入一种国内支付方式，MVP 商品固定为 10 元充值。
- 支付成功后写入余额账户和余额流水。
- 支付回调必须验签、幂等、校验金额。
- 控制台展示余额、充值记录和基础用量。

## P3：联网搜索

- Chat 页面增加“联网搜索”开关，默认关闭。
- API 支持 `tools: [{ "type": "tokenmesh:web_search" }]` 或 `web_search` 参数。
- 第一阶段只接一个搜索供应商，每次最多 5 条结果。
- Activity 记录搜索引擎、结果数、抓取页数、搜索成本和模型 token 成本。
- MVP 定价：每次联网搜索 ¥0.20 + 模型 token 费用。

## P4：火山引擎部署准备

- 数据库从 SQLite 迁移到火山引擎 RDS for PostgreSQL。
- 缓存和限流使用火山引擎 Redis。
- 异步任务和账务事件使用火山引擎消息队列产品。
- 准备生产环境变量清单、部署步骤和回滚方案。

## 建议下一步

下一步优先做 P0 安全与稳定性，尤其是 `JWT_SECRET` 生产必填、Chat 输入限制、上下文截断和上游错误脱敏。这些改动范围小，但能显著降低线上第一版风险。
